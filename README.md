# 📝 Notes App

Android-приложение для управления заметками с использованием кастомных View, собственной ViewGroup, архитектуры Clean Architecture, Jetpack Compose, Navigation Component, Coroutines и Flow.

## Скриншоты

Поддерживаются светлая и тёмная темы оформления.

## Возможности

### Работа с заметками

* Создание новых заметок
* Редактирование существующих заметок
* Отображение даты и времени создания
* Отметка заметок как важных
* Отметка заметок как прочитанных через Long Click
* Поиск по заголовкам заметок
* Валидация данных при создании и редактировании

---

## Кастомный NoteView

Реализирован собственный компонент `NoteView`, отображающий:

* Заголовок заметки
* Текст заметки
* Статус прочтения
* Время создания
* Индикатор важности

### Особенности

* Поддержка кастомных атрибутов через `attrs.xml`
* Поддержка собственных стилей:

  * `NoteStyle.NotRead`
  * `NoteStyle.Read`
* Ограничение текста двумя строками
* Fade-эффект для длинного текста
* Изменение внешнего вида в зависимости от состояния заметки

---

## Кастомный NoteStackView

Реализирована собственная `ViewGroup`, отображающая заметки в виде стопки.

### Возможности

* Ручное позиционирование элементов через `onLayout()`
* Наложение карточек друг на друга
* Автоматическая сортировка по дате создания
* Ограничение количества отображаемых элементов
* Поддержка настроек:

  * `stackSpacing`
  * `stackMaxVisible`
* Раскрытие и сворачивание стопки

### Анимация раскрытия

Реализована каскадная анимация:

* Элементы начинают движение поочередно
* Задержка старта: `i * 20ms`
* Продолжительность рассчитывается по формуле:

```text
duration = min(800ms, 200ms + count * 40ms)
```

* Используется кастомный `PathInterpolator`
* После завершения появляется кнопка сворачивания с анимацией:

  * Alpha: 0 → 1
  * Scale: 0.7 → 1.0

## Список заметок

Для отображения списка используется:

* RecyclerView
* Adapter Delegates
* ItemDecoration для разделения групп

### Логика отображения

Заметки группируются по дате без учета времени.

Структура списка:

```text
12.02.2026

[Important Note]
[Important Note]
[Note Stack]

11.02.2026

[Important Note]
[Note Stack]
```

### Правила

* Важные заметки отображаются отдельно
* Обычные заметки объединяются в NoteStackView
* Заголовки дат выводятся отдельными элементами списка

---

## Экран создания и редактирования

Экран реализован полностью на Jetpack Compose.

### Создание заметки

Поля:

* Заголовок (обязательное)
* Текст заметки
* Переключатель "Важно"

### Редактирование заметки

Дополнительно отображаются:

* Статус прочтения
* Дата создания

### Валидация

Если заголовок пустой:

```text
Необходимо заполнить
```

Если превышена максимальная длина:

```text
Слишком длинный заголовок, нужно сократить
```

Проверка выполняется через Flow без нагрузки на Main Thread.

---

## Навигация

Используется Jetpack Navigation.

Передача данных реализована через:

* NavArgs
* Parcelable

### Портретный режим

* Экран заметки открывается поверх списка
* Back возвращает к списку
* Повторный Back вызывает подтверждение выхода

### Ландшафтный режим

* Список занимает 50% экрана
* Экран заметки отображается рядом
* Повторное открытие заметок не создает дубликаты в back stack

```
com.example.notes
├── domain        # Бизнес-логика: сущности, интерфейсы репозиториев, use cases
├── data          # Реализация репозиториев, локальное хранилище (Room)
├── di            # DI-компоненты и модули
└── presentation  # UI: ViewModel, Compose-экраны, кастомные View, навигация
```

## Асинхронность

Используются:

* Kotlin Coroutines
* Kotlin Flow

### Реализовано

* Сохранение состояния при смене конфигурации
* Отсутствие дублирования фоновых операций

## Поиск

Реализован поиск по заголовкам заметок.

Особенности:

* debounce
* distinctUntilChanged
* фильтрация пустых запросов
* отсутствие лишних обращений к источнику данных

---

## Хранение данных

Все заметки сохраняются в постоянное хранилище.

### Дополнительно

* При первом запуске создаются демонстрационные заметки
* Пользовательские заметки сохраняются между запусками приложения

---

## Настройки

Добавлен экран настроек приложения.

Пользователь может изменять:

* расстояние между элементами стопки (`stackSpacing`)
* максимальное количество видимых элементов (`stackMaxVisible`)

---

## Shimmer Loading

Для первого запуска реализован экран загрузки с shimmer-анимацией.

Особенности:

* Показывается до получения данных
* Минимальная длительность отображения — 500 мс
* При последующих запусках не отображается
